MaxCompute(OPDS)官方教程

1. 什么是MaxCompute?

  1. MaxCompute提供了大量数据的存储以及分布式计算模型，即在阿里云开通MaxCompute之后可以存储海量数据以及通过其提供的计算模型进行数据分析。

适合多少数据的存储？

MaxCompute适用于100GB以上规模的存储及计算需求，最大可达EB级别。

主要提供的计算模型

MaxCompute支持SQL、MapReduce、UDF（Java/Python）、Graph、基于DAG的处理、交互式、内存计算、机器学习等计算类型及MPI迭代类算法。简化了企业大数据平台的应用架构。

系统架构

MaxCompute以数据为中心，内建多种计算模型和服务接口，满足广泛的数据分析需求。一切服务“开通”即用。

主要功能

  1. Tunnel：通过Tunnel提供海量数据传输服务（提供了java编程接口），通过该类可以实现高并发的离线数据上传下载服务。支持每天TB/PB级别的数据导入导出，特别适合于全量数据或历史数据的批量导入。

  2. DataHub服务：针对实时数据上传的场景，MaxCompute提供了延迟低、使用方便的DataHub服务，特别适用于增量数据的导入。

  3. 提供多种计算模型：

    1. SQL：以表的形式存储、操作数据
    2. UDF：用户自定义函数满足计算要求
    3. MapReduce编程模型：实现分布式计算
    4. SDK：它是MaxCompute提供给开发者的工具包，当前支持Java SDK及Python SDK。

MaxCompute与其他阿里云服务的集成使用

最重要的就是DataWorks，因为DataWorks控制台即为MaxCompute控制台。

DataWorks：DataWorks是基于MaxCompute计算和存储，提供工作流可视化开发、调度运维托管的一站式海量数据离线加工分析平台。在大数据平台中，DataWorks控制台即为MaxCompute控制台。

2. 程序员最常使用的模块

模块                说明
MapReduce        MaxCompute提供的Java MapReduce编程模型。您可以使用MapReduce提供的接口（Java API）编写                    MapReduce程序，处理MaxCompute中的数据。
Graph            一套面向迭代的图计算处理框架。使用图进行建模，图由点（Vertex）和边（Edge）组成，点和边包含                  权值（Value）。通过迭代对图进行编辑、演化，最终得出结果。
Tunnel            您可以使用Tunnel服务向MaxCompute批量上传离线数据或从MaxCompute下载离线数据。
Java SDK        向开发者提供的Java接口。
Python SDK        向开发者提供的Python接口。

3. MaxCompute术语表

术语表与基本概念介绍链接

##　重要术语介绍

  1. 项目：MaxComputed的基本组织单元，类似传统数据库中的database、Schema的概念，可以实现用户隔离。项目中包含了多个对象：表、资源、函数、实例等。

  2. 表：数据存储单元，分为内部表和外部表（MaxCompute并不真的保存外部表，仅仅保存其元数据）。

注意： MaxCompute中的表包含了普通表和分区表（即表中存在分区的概念，下面介绍。）

  1. 分区：创建表时指定表内的一个或者某几个字段作为分区列。分区表实际就是对应分布式文件系统上的独立的文件夹，一个分区对应一个文件夹，文件夹下是对应分区所有的数据文件。

通俗理解：根据指定字段的值将数据存储到不同的文件夹下，示例如下图。

  1. 资源：我们自己写的代码上传到MaxCompute即为资源，因此自定义函数（UDF）或MapReduce的使用都需要依赖资源。
  2. 任务：见链接文档。
  3. 实例：在MaxCompute中，SQL、Spark和Mapreduce任务在执行时会被实例化，以MaxCompute实例（下文简称为实例或Instance）的形式存在。实例会经历运行（Running）和结束（Terminated）两个阶段。
